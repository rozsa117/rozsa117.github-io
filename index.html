CryptoStreams

CryptoStreams is an open source tool for output data generation of specific, parametrized and possibly round-reduced, cryptographic functions, developed at CRoCS lab of Masaryk University. On this web page, you can find the information about all the function included in the project. The functions are analyzed with four different statistical batteries STS NIST, Dieharder, TestU01, and BoolTest for numerous data generation strategies and lengths.

Randomness Testing Toolkit (RTT) is a tool that simplifies the execution of multiple statistical batteries (now STS NIST, Dieharder, TestU01) via a web interface and provides easy-to-understand interpretation of the results.

We used RTT to analyze a large number of cryptographic functions from CryptoStreams project with the results listed below.
RTT currently implements support for the following statistical tools:

  • NIST Statistical Test Suite (NIST STS) - the classic and well-known suite of tests required for NIST FIPS140-2 devices according to NIST SP 800-22.

  • Dieharder: A Random Number Test Suite - improved testing suite containing additional tests atop of selected ones from NIST STS.

  • TestU01 - extensive and powerful testing suite, suitable for testing larger data streams. Typically more sensitive than NIST STS and Dieharder.

  • BoolTest - test based on efficient search for Boolean functions (to be added soon).

  • Counter (CTR)
    • The input blocks are generated as a counter value starting from 0. Intuitively, the higher bits are set to zero while the low bits are incremented. The goal is to test the diffusion and masking property of the analyzed function in case of low entropy inputs and very small changes between the subsequent inputs.

  • Low Hamming weight (HW)
    • The input blocks are generated to have minimum Hamming weight possible which still not repeated (the actual Hamming weight depends on the required output size). E,g., if a 100 MB of output data is required, and the length of the input block is 128 bits, the Hamming weight is set to 4 (number of different input blocks with Hamming weight 4 is equal to 128 over 4, and the length of the output block is 16 bytes => ~ 162 MB maximum). If more than 162 MB is required, Hamming weight 5 (or more) needs to be used. The goal is to test the masking property in case of inputs with very low entropy.

  • Strict Avalanche Criterion (SAC)
    • The input blocks are generated as a pair where the first block is random and the second one equals to the first block except for single bit flip. The output blocks are then the concatenation of the output blocks. The goal is to test the diffusion property of the analyzed function.

  • Random-plaintext-ciphertext (RPC)
    • The input blocks are generated at random, and the output is a concatenation of the input and its corresponding output block (p_i || f(pi)). The goal is to directly analyze function's input-output correlation for the analyzed function (similar to linear cryptoanalysis).
